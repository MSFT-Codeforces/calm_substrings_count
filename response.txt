We count how many substrings of a given string are "calm"—meaning no single letter appears in more than half of the substring. The solution avoids checking every substring by counting the non-calm ones and subtracting from the total.

**What we're solving**

You have a string $s$ of length $n$ with lowercase letters. For each substring $s[l..r]$, call its length $m = r - l + 1$ and let $\text{freq}(x)$ be how many times letter $x$ appears in it. The substring is calm if no letter appears strictly more than half the time:
$$
\forall x \in \{a,\dots,z\},\ \text{freq}(x) \le \frac{m}{2}.
$$
We need the number of calm substrings.

**Idea of the approach**

Instead of testing each substring, we count substrings that are *not* calm and subtract that from the total.

The total number of substrings is $\frac{n(n+1)}{2}$, which we can compute directly. A substring fails to be calm when some letter appears strictly more than half the time; we can call that letter the "overlord" of that substring. At most one letter can be an overlord for a given substring (if two letters each exceeded half, their counts would add up to more than the length). So we can count non-calm substrings by going through each of the 26 letters, counting substrings where that letter is the overlord, and adding those counts. No double counting.

For a fixed letter (e.g. 'a'), replace every character in the string with +1 if it's 'a' and −1 otherwise. A substring where 'a' appears in more than half the positions is exactly a substring whose sum is strictly positive. So counting non-calm substrings with 'a' as overlord is the same as counting subarrays with positive sum in this +1/−1 array.

Using prefix sums: build the running prefix sum over that array. A subarray from $l$ to $r$ has positive sum exactly when the prefix at $r$ is strictly greater than the prefix at $l-1$. So for each position $r$, we need how many earlier prefix values are strictly smaller than the current one.

The prefix sums lie in a range of size $O(n)$, so we can shift them to positive indices and use a Fenwick tree. As we scan left to right, we keep in the tree how many times each prefix-sum value has appeared. For each new prefix we ask: how many previous values are strictly less than this one? That's one query and one update, both $O(\log n)$. We do this for each of the 26 letters (resetting the tree each time), sum the non-calm counts, then subtract from the total number of substrings to get the number of calm substrings.

**Code**
```cpp
#include <iostream>
#include <string>
#include <vector>

/**
 * @brief Fenwick Tree (Binary Indexed Tree) supporting point updates and prefix-sum queries.
 */
class FenwickTree {
public:
    /**
     * @brief Creates a Fenwick tree and initializes it to a given size with all zeros.
     *
     * @param treeSize One-based maximum index supported by the tree.
     */
    explicit FenwickTree(int treeSize) {
        resetTree(treeSize);
    }

    /**
     * @brief Resets the tree to a given size and clears all stored values.
     *
     * @param treeSize One-based maximum index supported by the tree.
     */
    void resetTree(int treeSize) {
        this->treeSize = treeSize;
        binaryIndexedTree.assign(treeSize + 1, 0LL);
    }

    /**
     * @brief Adds a value to a single index.
     *
     * @param oneBasedIndex One-based index to update.
     * @param delta Value to add at the index.
     */
    void addValue(int oneBasedIndex, long long delta) {
        while (oneBasedIndex <= treeSize) {
            binaryIndexedTree[oneBasedIndex] += delta;
            oneBasedIndex += oneBasedIndex & -oneBasedIndex;
        }
    }

    /**
     * @brief Computes the prefix sum from index 1 to oneBasedIndex.
     *
     * @param oneBasedIndex One-based index (inclusive).
     * @return Sum of values in range [1..oneBasedIndex].
     */
    long long getPrefixSum(int oneBasedIndex) const {
        long long result = 0;
        while (oneBasedIndex > 0) {
            result += binaryIndexedTree[oneBasedIndex];
            oneBasedIndex -= oneBasedIndex & -oneBasedIndex;
        }
        return result;
    }

private:
    int treeSize = 0;
    std::vector<long long> binaryIndexedTree;
};

/**
 * @brief Reads input, counts calm substrings, and prints the result.
 *
 * @return Exit code 0 on successful execution.
 */
int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int stringLength = 0;
    std::string inputString;
    std::cin >> stringLength >> inputString;

    const long long totalSubstrings = 1LL * stringLength * (stringLength + 1) / 2;

    // Prefix sums lie in [-n, n]. Shift them to be valid 1-based Fenwick indices.
    const int prefixSumOffset = stringLength + 2;
    const int fenwickTreeSize = 2 * stringLength + 5;

    FenwickTree fenwickTree(fenwickTreeSize);
    long long totalOverlordSubstrings = 0;

    for (char targetChar = 'a'; targetChar <= 'z'; ++targetChar) {
        fenwickTree.resetTree(fenwickTreeSize);

        int prefixSumValue = 0;
        fenwickTree.addValue(prefixSumValue + prefixSumOffset, 1);

        long long positiveSumSubarrays = 0;

        for (int positionIndex = 0; positionIndex < stringLength; ++positionIndex) {
            prefixSumValue += (inputString[positionIndex] == targetChar) ? 1 : -1;
            const int mappedPrefixSumIndex = prefixSumValue + prefixSumOffset;

            positiveSumSubarrays += fenwickTree.getPrefixSum(mappedPrefixSumIndex - 1);
            fenwickTree.addValue(mappedPrefixSumIndex, 1);
        }

        totalOverlordSubstrings += positiveSumSubarrays;
    }

    const long long calmSubstrings = totalSubstrings - totalOverlordSubstrings;
    std::cout << calmSubstrings;
    return 0;
}
```

**How the code works**

The FenwickTree class is a standard Binary Indexed Tree. `addValue(i, delta)` adds `delta` at index `i`; `getPrefixSum(i)` returns the sum from 1 to `i`. Both walk the tree in $O(\log n)$. `resetTree` re-initializes everything to zeros.

The program reads the string and computes `totalSubstrings` as $n(n+1)/2$ in 64-bit to avoid overflow. Prefix sums live in $[-n, n]$, so we use `prefixSumOffset = n + 2` to map them to positive 1-based indices; the tree size is $2n+5$ to cover that range with some slack.

For each letter from 'a' to 'z', we count substrings where that letter is the overlord. We reset the Fenwick tree and insert the initial prefix sum $P_0 = 0$ at the offset index before scanning. For each character we add +1 if it matches the current letter and −1 otherwise, update the prefix sum, then query how many previous prefix values are strictly less than the current one—that's exactly the number of substrings ending here where this letter dominates. We add that to the running count and insert the current prefix into the tree. After all 26 letters, we subtract the total non-calm count from the total number of substrings and print.

**Example: $s = \texttt{"aab"}$, $n = 3$**

The six substrings are "a", "a", "b", "aa", "ab", and "aab". Only "ab" is calm (one of each letter, and $1 \le 2/2$). So the answer is 1.

We have `totalSubstrings` $= 6$, `prefixSumOffset` $= 5$, `fenwickTreeSize` $= 11$.

For targetChar 'a' the array is $[+1,+1,-1]$. We start with prefix 0 in the tree. At $i=0$ (first 'a') prefix becomes 1; there is 1 earlier value smaller than 1 (the initial 0), so we add 1. At $i=1$ prefix is 2; there are 2 earlier values smaller than 2 (0 and 1), so we add 2. At $i=2$ prefix is 1; there is 1 earlier value smaller than 1 (0), so we add 1. So 'a' is overlord in 4 substrings: "a", "a", "aa", "aab".

For 'b' the array is $[-1,-1,+1]$. Prefixs go 0, −1, −2, −1. Only when we reach the last position do we have a previous prefix (−2) smaller than the current (−1), giving 1 substring where 'b' is overlord ("b"). So `totalOverlordSubstrings` $= 4 + 1 = 5$. For 'c' through 'z' the array is all −1, so prefix sums only decrease and no subarray has positive sum; they contribute 0.

So `calmSubstrings` $= 6 - 5 = 1$, which is correct. For the example $s = \texttt{"zzzzazzzzzz"}$ with $n = 11$, the same idea gives 66 total substrings, 64 non-calm (mostly dominated by 'z'), and 2 calm—the two length-2 windows "za" and "az" around the single 'a'.

**Complexity**

Time is $O(26 \cdot n \log n)$: for each letter we do $n$ Fenwick operations. Space is $O(n)$ for the tree.
