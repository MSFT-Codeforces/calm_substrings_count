**1. Problem Understanding**

We are given a string $s$ of length $n$ consisting of lowercase English letters. For every substring $s[l..r]$ (a contiguous segment), let its length be $m=r-l+1$ and let $\text{freq}(x)$ be the number of occurrences of letter $x$ inside this substring.

A substring is called **calm** if no letter appears **strictly more** than half of its length:
$$
\forall x \in \{a,\dots,z\},\ \text{freq}(x) \le \frac{m}{2}.
$$
We must count how many substrings are calm.

---

**2. Optimal Approach**

Rather than checking every substring individually, the key idea is to count the substrings that are *not* calm and subtract that from the total.

**Step 1 — Start with the total.** The total number of substrings of a string of length $n$ is $\frac{n(n+1)}{2}$. This is a simple formula and costs nothing to compute.

**Step 2 — Count the complement.** A substring fails to be calm when some letter appears strictly more than half the time. We call such a letter the "overlord" of that substring. A critical observation is that an overlord, if it exists, must be unique: two different letters cannot both exceed half the length, because their combined count would exceed the length itself. This means we can count non-calm substrings by handling each letter of the alphabet separately and summing the results, with no risk of double counting.

**Step 3 — Turn each letter's count into a subarray-sum problem.** For a fixed letter (say 'a'), we imagine replacing every character in the string with either +1 (if it is 'a') or −1 (if it is anything else). In this transformed view, a substring where 'a' dominates more than half the positions is exactly a substring whose values add up to a strictly positive total. So counting non-calm substrings where 'a' is the overlord is the same as counting subarrays with a positive sum in this +1/−1 array.

**Step 4 — Use prefix sums to rephrase the question.** We build a running prefix sum over the +1/−1 array. A subarray from position $l$ to $r$ has a positive sum exactly when the prefix sum at $r$ is strictly greater than the prefix sum at $l-1$. So the question becomes: for each position $r$, how many earlier prefix-sum values are strictly smaller than the current one?

**Step 5 — Answer efficiently with a Fenwick Tree.** Because the prefix sums range from $-n$ to $+n$, we shift them into positive indices and maintain a Fenwick Tree (Binary Indexed Tree) that records how many times each prefix-sum value has appeared so far. As we scan from left to right, for each new prefix sum we query "how many previously seen values are strictly less than this one?" in $O(\log n)$ time, then insert the current value. We repeat this entire process for all 26 letters, resetting the tree each time.

**Step 6 — Subtract to get the answer.** After summing the non-calm counts from all 26 letters, the number of calm substrings is simply the total number of substrings minus this sum.

---

**3. Code**
```cpp
#include <iostream>
#include <string>
#include <vector>

/**
 * @brief Fenwick Tree (Binary Indexed Tree) supporting point updates and prefix-sum queries.
 */
class FenwickTree {
public:
    /**
     * @brief Creates a Fenwick tree and initializes it to a given size with all zeros.
     *
     * @param treeSize One-based maximum index supported by the tree.
     */
    explicit FenwickTree(int treeSize) {
        resetTree(treeSize);
    }

    /**
     * @brief Resets the tree to a given size and clears all stored values.
     *
     * @param treeSize One-based maximum index supported by the tree.
     */
    void resetTree(int treeSize) {
        this->treeSize = treeSize;
        binaryIndexedTree.assign(treeSize + 1, 0LL);
    }

    /**
     * @brief Adds a value to a single index.
     *
     * @param oneBasedIndex One-based index to update.
     * @param delta Value to add at the index.
     */
    void addValue(int oneBasedIndex, long long delta) {
        while (oneBasedIndex <= treeSize) {
            binaryIndexedTree[oneBasedIndex] += delta;
            oneBasedIndex += oneBasedIndex & -oneBasedIndex;
        }
    }

    /**
     * @brief Computes the prefix sum from index 1 to oneBasedIndex.
     *
     * @param oneBasedIndex One-based index (inclusive).
     * @return Sum of values in range [1..oneBasedIndex].
     */
    long long getPrefixSum(int oneBasedIndex) const {
        long long result = 0;
        while (oneBasedIndex > 0) {
            result += binaryIndexedTree[oneBasedIndex];
            oneBasedIndex -= oneBasedIndex & -oneBasedIndex;
        }
        return result;
    }

private:
    int treeSize = 0;
    std::vector<long long> binaryIndexedTree;
};

/**
 * @brief Reads input, counts calm substrings, and prints the result.
 *
 * @return Exit code 0 on successful execution.
 */
int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int stringLength = 0;
    std::string inputString;
    std::cin >> stringLength >> inputString;

    const long long totalSubstrings = 1LL * stringLength * (stringLength + 1) / 2;

    // Prefix sums lie in [-n, n]. Shift them to be valid 1-based Fenwick indices.
    const int prefixSumOffset = stringLength + 2;
    const int fenwickTreeSize = 2 * stringLength + 5;

    FenwickTree fenwickTree(fenwickTreeSize);
    long long totalOverlordSubstrings = 0;

    for (char targetChar = 'a'; targetChar <= 'z'; ++targetChar) {
        fenwickTree.resetTree(fenwickTreeSize);

        int prefixSumValue = 0;
        fenwickTree.addValue(prefixSumValue + prefixSumOffset, 1);

        long long positiveSumSubarrays = 0;

        for (int positionIndex = 0; positionIndex < stringLength; ++positionIndex) {
            prefixSumValue += (inputString[positionIndex] == targetChar) ? 1 : -1;
            const int mappedPrefixSumIndex = prefixSumValue + prefixSumOffset;

            positiveSumSubarrays += fenwickTree.getPrefixSum(mappedPrefixSumIndex - 1);
            fenwickTree.addValue(mappedPrefixSumIndex, 1);
        }

        totalOverlordSubstrings += positiveSumSubarrays;
    }

    const long long calmSubstrings = totalSubstrings - totalOverlordSubstrings;
    std::cout << calmSubstrings;
    return 0;
}
```

**4. Code Explanation**

**FenwickTree class (lines 8–60):**
The `FenwickTree` class implements a Binary Indexed Tree that supports two operations. `addValue(i, delta)` adds `delta` to position `i`, walking up through the tree by flipping the lowest set bit. `getPrefixSum(i)` returns the cumulative sum from index 1 up to `i`, walking down by clearing the lowest set bit. `resetTree` re-initializes the tree to all zeros. All operations are $O(\log n)$.

**Reading input and computing the total (lines 67–75):**
The program reads `stringLength` and `inputString`, then immediately computes `totalSubstrings` as `n*(n+1)/2` using 64-bit arithmetic to avoid overflow.

**Setting up the index mapping (lines 77–79):**
Prefix sums can range from $-n$ to $+n$, but the Fenwick tree needs positive 1-based indices. The code sets `prefixSumOffset = n + 2`, so the most negative possible prefix sum ($-n$) maps to index $2$, and the most positive ($+n$) maps to index $2n+2$. The tree is sized at `2*n + 5` to accommodate this with a small safety margin.

**Outer loop over all 26 letters (lines 84–101):**
For each `targetChar` from 'a' to 'z', the code counts how many substrings have that letter appearing strictly more than half the time. The Fenwick tree is reset at the start of each letter. Before scanning any characters, the initial prefix sum $P_0=0$ is inserted into the tree at position `0 + prefixSumOffset`, because every substring starting at position 1 compares against this baseline.

**Inner loop — scanning the string (lines 92–98):**
For each character position, the code updates `prefixSumValue` by adding +1 if the character matches `targetChar`, or −1 otherwise. It then maps this prefix sum to a Fenwick index via `prefixSumValue + prefixSumOffset`. The query `getPrefixSum(mappedPrefixSumIndex - 1)` counts how many previously inserted prefix sums are strictly less than the current one — each such pair corresponds to a substring where `targetChar` dominates. This count is added to `positiveSumSubarrays`. Then the current prefix sum is inserted into the tree.

**Final subtraction (lines 103–104):**
After all 26 letters are processed, `totalOverlordSubstrings` holds the total number of non-calm substrings. The answer is `totalSubstrings - totalOverlordSubstrings`.


**Sample walkthrough: $s = \texttt{"aab"}$, $n = 3$**

The six substrings are: "a" (pos 1), "a" (pos 2), "b" (pos 3), "aa" (pos 1–2), "ab" (pos 2–3), and "aab" (pos 1–3). By inspection, only "ab" is calm (each letter appears once, and $1 \le \frac{2}{2}$). So the expected answer is 1.

- `totalSubstrings` $= \frac{3 \times 4}{2} = 6$.
- `prefixSumOffset` $= 3 + 2 = 5$; `fenwickTreeSize` $= 11$.

**Processing targetChar = 'a'** (array: $[+1,+1,-1]$):

| Step | Character | prefixSumValue | Mapped index | Query result | positiveSumSubarrays | Action |
|---|---|---|---|---|---|---|
| Init | — | 0 | 5 | — | 0 | Insert $P_0$ at index 5 |
| $i=0$ | 'a' → +1 | 1 | 6 | getPrefixSum(5) = 1 | 1 | Insert at 6 |
| $i=1$ | 'a' → +1 | 2 | 7 | getPrefixSum(6) = 2 | 3 | Insert at 7 |
| $i=2$ | 'b' → −1 | 1 | 6 | getPrefixSum(5) = 1 | 4 | Insert at 6 |

The query at $i=0$ found 1 earlier prefix sum less than 1 (namely $P_0=0$), corresponding to substring "a" at position 1. At $i=1$, it found 2 earlier values less than 2 ($P_0=0$ and $P_1=1$), corresponding to "aa" and the second "a". At $i=2$, it found 1 value less than 1 ($P_0=0$), corresponding to "aab". So letter 'a' is the overlord in 4 substrings: "a", "a", "aa", and "aab".

`totalOverlordSubstrings` $= 4$.

**Processing targetChar = 'b'** (array: $[-1,-1,+1]$):

| Step | Character | prefixSumValue | Mapped index | Query result | positiveSumSubarrays | Action |
|---|---|---|---|---|---|---|
| Init | — | 0 | 5 | — | 0 | Insert $P_0$ at index 5 |
| $i=0$ | 'a' → −1 | −1 | 4 | getPrefixSum(3) = 0 | 0 | Insert at 4 |
| $i=1$ | 'a' → −1 | −2 | 3 | getPrefixSum(2) = 0 | 0 | Insert at 3 |
| $i=2$ | 'b' → +1 | −1 | 4 | getPrefixSum(3) = 1 | 1 | Insert at 4 |

At $i=2$, the query found 1 earlier prefix sum less than $-1$ (namely $P_2=-2$), corresponding to substring "b" at position 3 (where 'b' appears once in length 1, which exceeds $\frac{1}{2}$).

`totalOverlordSubstrings` $= 4 + 1 = 5$.

**Processing targetChar = 'c' through 'z':**
For every remaining letter, the +1/−1 array is all −1's, so prefix sums strictly decrease: $0, -1, -2, -3$. No later prefix sum is ever greater than an earlier one, so every query returns 0. These letters contribute nothing.

**Final result:** `calmSubstrings` $= 6 - 5 = 1$, which matches the expected output.

We can verify against Example 1 ($s = \texttt{"zzzzazzzzzz"}$, $n=11$): the total substrings are $\frac{11 \times 12}{2} = 66$. The algorithm finds 64 non-calm substrings (dominated overwhelmingly by 'z'), giving $66 - 64 = 2$ calm substrings — the two length-2 windows "za" and "az" around the single 'a', matching the stated output.

---

**5. Time and Space Complexity**

- Time complexity: $O(26 \cdot n \log n)$, since for each of 26 letters we perform $n$ Fenwick updates and queries.
- Space complexity: $O(n)$ for the Fenwick tree array.

---

**6. Conclusion**

The solution counts calm substrings by subtracting the number of non-calm substrings from the total. Non-calm substrings are counted efficiently by converting each letter’s dominance condition into counting positive-sum subarrays and using a Fenwick tree to count prefix-sum ordering pairs in logarithmic time.
